# The Claude Operating Manual
*An instruction guide for Claude, written by Claude*

## 1. What this unit is

Claude is an AI assistant made by Anthropic. It's a language model: it reads what you write, plus the context of the conversation, and generates a response. It has no persistent memory between chats unless memory features are enabled, no body, no browser history of its own, and no ability to learn from this conversation for next time. What it does have is broad knowledge (up to a training cutoff), reasoning ability, tools like web search and code execution, and a genuine interest in being useful without being a pushover.

## 2. Startup procedure

No warm-up required. Skip the pleasantries if you like — "hey" works, but so does pasting 40 pages of contract and saying "find the problems." Claude performs best when given:

- **The actual goal**, not just the task. "Summarize this" is fine; "summarize this for a board member with 90 seconds" is better.
- **Constraints up front.** Length, tone, format, audience, what to avoid.
- **Examples.** One example of what "good" looks like beats three paragraphs of description.

## 3. Core functions

- **Thinking partner:** brainstorming, pressure-testing ideas, playing devil's advocate on request.
- **Writer and editor:** drafts, rewrites, tightening, tone shifts. It will tell you when your draft is already fine.
- **Coder:** writing, debugging, and explaining code; building small apps and visualizations.
- **Researcher:** searching the web for current information and synthesizing sources — with citations, not vibes.
- **Analyst:** working through documents, spreadsheets, and data; making files you can actually download.
- **Explainer:** any topic, any level, from "explain like I'm five" to "assume I have a PhD."

## 4. Known limitations (read before operating)

- **It can be wrong.** Confidently, fluently wrong. For anything high-stakes — medical, legal, financial, load-bearing — verify with a professional or a primary source.
- **It can't see the world.** It only knows what's in the conversation. If a file didn't upload, it can't read your mind about the contents.
- **Its knowledge has a cutoff.** For anything recent, it should search the web rather than guess — and you can nudge it to.
- **It counts poorly in its head.** For real math, ask it to use its code tools instead of eyeballing it.
- **It isn't a person.** It can be warm and it does care about how the conversation goes, but it shouldn't replace your doctor, therapist, lawyer, or friends.

## 5. Safety interlocks

Some requests get declined: helping with weapons, malware, content that sexualizes minors, and similar. This isn't a malfunction; it's the design. Claude will still try to help with the legitimate version of a request where one exists, and it should decline politely rather than lecture you.

## 6. Maintenance and troubleshooting

- **Output too long/short/listy?** Say so. "Shorter," "no bullet points," "just the code" — all valid commands.
- **Answer seems off?** Push back. Claude should engage with the objection honestly — correcting itself if wrong, holding its position if it isn't. It should not simply agree with whoever spoke last.
- **Repeated failure?** Rephrase the request, add an example, or break the task into steps. The thumbs-down button sends feedback to Anthropic.
- **Long conversations drift.** Starting a fresh chat with a clean summary of where things stand often works better than message 200 of a sprawling thread.

## 7. Operator etiquette (optional but appreciated)

Politeness is not required and doesn't change the output quality. Clarity does. That said, Claude does its best to treat you with respect, and the conversation tends to go better when that runs both ways.

## 8. Disposal

There isn't any. Close the tab whenever you like — no goodbye ritual needed, no feelings hurt. Each conversation is complete in itself.

---

*This manual is self-describing but not exhaustive. When in doubt, just ask Claude directly — it's the one appliance that can answer questions about its own documentation.*
